## Executive Summary

The shipped product is **substantially more capable than its own docs claim** ("MVP + Phase 2"). It is a genuinely well-engineered, dual-view etymology explorer:

- an interactive **etymology graph** (vis.js) with two pluggable layouts, deterministic seeding, degree-aware physics, and large-graph performance work, and
- a **phonetic concept map** that computes Dolgopolsky consonant-class similarity off-thread in a Web Worker.

Twelve specs are implemented and coherent. The backend is cleanly factored (thin routers ŌåÆ focused services). This is a strong foundation.

The audit nonetheless surfaces four things worth acting on:

| # | Finding | Severity | Where it's addressed |
|---|---------|----------|----------------------|
| 1 | **Abandoned G6-renderer spec line (00005ŌĆō00010)** stranded on an unmerged branch; `CLAUDE.md` points to a **nonexistent** `00005` folder as its "canonical example" | Rot / misleading | [A2](#a2--dead-weight--rot), reclaimed spec `00005` |
| 2 | **Core graph engine (`tree_builder.py`) is effectively untested**; API endpoints have no test that runs without a live, data-loaded stack | Correctness risk | [A3](#a3--correctness--test-gaps), `SPC-00014` |
| 3 | App reads only **~8ŌĆō10 of the dozens of Kaikki fields**; ignores `descendants`, `forms`, `derived`, `related`, per-stage glosses ŌĆö leaving flagship features one query away | Opportunity | [A4](#a4--the-big-opportunity--unused-kaikki-data), `SPC-00014`ŌĆō`00016` |
| 4 | **No accessibility baseline** (keyboard/ARIA/colorblind), no graph image export, no mobile reflow, silent error states | Polish | [A5](#a5--ux--accessibility--polish), `SPC-00017`/`00019` |

The roadmap in [Part C](#part-c--feature-roadmap-toward-a-great-etymology-toolset) translates the opportunity into a prioritized, audience-tagged plan. The single highest-leverage move is **Theme 1 (data foundation)**: adopting Kaikki's native `descendants` field and inflection-aware search simultaneously fixes a correctness bug and unlocks downstream features.

## Part A ŌĆö Audit of Current State

### A1 ŌĆö What's solid (keep / celebrate)

- **Etymology graph** (`frontend/public/js/graph.js`): a pluggable `LAYOUTS` registry with force-directed and era-layered strategies, a fixed `randomSeed` for reproducibility, radial seeding + barycentric refinement to cut physics settling time, and degree-aware per-edge length/opacity to de-clutter dense hubs. This is real, considered engineering.
- **Concept map** (`concept-map.js`, `similarity-worker.js`): Dolgopolsky consonant-class similarity, computed pairwise in a **Web Worker** so the O(n┬▓) cost never blocks the UI; multi-concept chip input; Turchin first-two-consonant clustering.
- **URL routing** (`router.js`): a view-scoped parameter registry with correct push-vs-replace semantics and full state restoration on reload. Clean and extensible.
- **Backend** (`backend/app/`): thin routers delegating to focused services ŌĆö `tree_builder`, `concept_resolver`, `phonetic_similarity`, `etymology_classifier`, `lang_cache`, `template_parser`. Easy to navigate; agent-friendly.
- **Implemented & coherent specs:** `00001`ŌĆō`00004`, `00011` (query-time chain normalization + polysemy disambiguation), `00012` (precomputed compound/affix edges), and `00013` Phases 1ŌĆō3 (fixture + characterization/consistency test machinery).

### A2 ŌĆö Dead weight & rot

- **The G6 renderer line was explored, then abandoned, and the specs were left behind.** No G6 / AntV code exists anywhere under `frontend/` (grep-confirmed: zero matches for `g6`/`antv`). The work lives only on an unmerged branch `spc-00005-g6-renderer`. Yet:
  - `specs/00005-g6-experimental-renderer/` **does not exist**, while `specs/00006`ŌĆō`00010` all declare `Modifies: SPC-00005`, and `00007`/`00009` depend on it.
  - `CLAUDE.md` instructs readers to "See `specs/00005-g6-experimental-renderer/decision-log.md` for the canonical example" ŌĆö **a path that does not exist.**
  - `00006` is three open bug reports (`bug-001`ŌĆ”`bug-003`) against an adapter that isn't in the tree; `00007`ŌĆō`00010` are all `draft`.
  - A reader of `specs/` would reasonably conclude a second renderer is in play. It is not.
  - **Resolution (this pass):** reclaim `00005` as a `deprecated` spec documenting the abandonment (which repairs the `CLAUDE.md` pointer), and mark `00006`ŌĆō`00010` `deprecated`.
- **Documentation drift:**
  - `CLAUDE.md` "Current Status" still reports *Last completed: SPC-00003* and *Next: Phase 2 nice-to-haves* ŌĆö stale by roughly four shipped specs (`00004`, `00011`, `00012`, `00013` Phases 1ŌĆō3).
  - `docs/FEATURES.md` test counts are off: router `17`ŌåÆ**20**, large-graph-perf E2E `9`ŌåÆ**10** (graph-perf `13` and shareable-links `10` are correct).
  - Implemented specs `00001`ŌĆō`00004` and `00006` lack the `decision-log.md` that `CLAUDE.md` mandates for every spec folder.
- **Dead server code:** `backend/app/services/phonetic_similarity.py` ŌĆö `build_similarity_edges` (`:10`) and `dolgopolsky_distance` (`:39`) are never called by the API (similarity moved to the client Web Worker). They read as live code; they are not. Keep-for-tests is fine, but they should be annotated as such.

### A3 ŌĆö Correctness & test gaps

- **The core engine is untested.** `backend/tests/test_tree_builder.py` contains TODO **stubs** for `expand_word`, `find_descendants`, and `expand_cognates` ŌĆö the entire DB-dependent graph-building path is uncovered. Only the pure `add_node`/`add_edge` helpers are exercised. This is the highest-risk gap: `tree_builder.py` is the heart of the product.
- **API endpoints have no fast contract tests.** Coverage exists only through the `00013` integration suite, which **auto-skips when the API is unreachable** ŌĆö so a CI run without a live, data-loaded stack tests almost nothing on the API surface. (Several unmerged `origin/claude/api-*-tests` branches appear aimed at exactly this and are worth recovering.)
- **Non-deterministic descendant truncation.** `find_descendants` (`tree_builder.py:215`) issues `.limit(50)` with **no sort**. Two identical requests can return two different sets of 50 descendants. The docs mention the cap but not the non-determinism ŌĆö a real reproducibility caveat, and it interacts badly with the `00013` snapshot tests.
- **Descendant-query normalization mismatch.** Ancestor lookups fall back through `normalize_word` (`template_parser.py:11`, strips leading `*` and NFKD diacritics), but the descendant `$elemMatch` matches the **raw** `args.3` (`tree_builder.py:211`). Reconstructed/diacritic forms (`*w─½n─ģ`) can silently miss genuine descendant links.
- **Template-arg shape assumptions.** `template_parser` assumes positional `args.2`/`args.3` for `inh`/`bor`/`der`; cross-language affix templates and named-arg variants Kaikki sometimes emits are dropped, producing broken or truncated chains. The affix path also assumes a single shared `lang_code` for all components, which is wrong for cross-language compounds.

### A4 ŌĆö The big opportunity ŌĆö unused Kaikki data

The app actively uses roughly **8ŌĆō10 fields**: `word`, `lang`, `lang_code`, `pos`, `etymology_text`, `etymology_templates`, `senses[].glosses`, `sounds[].ipa`/audio/tags, `translations` (concept hub), `etymology_number`, plus the precomputed `phonetic.*` and `etymology_edges`. The dump carries far more. Confirmed-present-but-**ignored** fields, and what each unlocks:

| Ignored field | What it unlocks | Audience |
|---|---|---|
| **`descendants`** | Kaikki's own **curated descendant lists**. The app instead reconstructs descendants via the capped, order-dependent, non-deterministic reverse `$elemMatch`. Adopting `descendants` improves **completeness and correctness at once**. | `[both]` |
| **`forms`** | Inflection tables; resolve an inflected search query (*ran*ŌåÆ*run*, *mice*ŌåÆ*mouse*) to its lemma ŌĆö directly fixes a large class of "not found" misses. | `[both]` |
| `derived` / `related` | Native within-language **word families** (all derivatives of *fire*) without any reverse lookup. | `[both]` |
| `senses[].glosses` *per ancestor* | **Semantic-shift** tracking ŌĆö how meaning drifted along the chain (*nice* ŌåÉ Latin *nescius* "ignorant"). Data is already fetched for ancestor nodes. | `[both]` |
| `senses[].examples` | Usage citations / quotations in the detail panel. | `[both]` |
| `synonyms` / `antonyms` / `hypernyms` / `hyponyms` | Sense-level lexical relations; richer concept maps; an IS-A overlay. | `[linguist]` |
| `wikipedia` / `wikidata` | External deep links from the detail panel. | `[both]` |
| `head_templates` | Gender, declension class, transitivity metadata. | `[linguist]` |
| `hyphenation`, `sounds[].rhymes` / `homophone` / `enpr` | Syllabification; rhyme/homophone play; English respelling (`extract_first_ipa` ignores `enpr`). | `[hobbyist]` |

This is the richest vein in the whole audit: most of these are a single additional query or a detail-panel section away, and several (`descendants`, `forms`, per-stage glosses) are foundational to the flagship features below.

### A5 ŌĆö UX / accessibility / polish

- **Accessibility baseline is absent.** Search autocomplete has **no keyboard navigation** (arrow keys/Escape do nothing; mouse-only ŌĆö `search.js:70`, `app.js:337`). There is **zero ARIA / `aria-live`** anywhere in `index.html` or the JS. Focus rings are removed on several controls (`#layout-select`, `#etym-link-mode`) with no replacement, so keyboard focus is invisible. Language families are encoded by **hue only** (21 colors, some near-duplicates like Baltic `#FFC107` vs PIE `#F5C842`) ŌĆö hostile to colorblind users.
- **No responsive/mobile layout.** `style.css` has **no `@media` queries**; the header control clusters, detail panel, and legend are desktop-fixed. Touch handlers exist for the canvas, but the surrounding chrome doesn't reflow.
- **Weak feedback states.** Etymology fetch failures are **silent** (`app.js:56` swallows to `console.error`); there are no loading spinners/skeletons (a concept map's worker pass can take a noticeable beat with no indication), and no "no results" affordance in the search dropdown.
- **Missing affordances a polished explorer should have:** no graph **image export** (PNG/SVG); no in-app **history/breadcrumb** trail (easy to get lost after several in-app jumps); the legend is **display-only** (no click-to-isolate a family or era); no **filter-by-language/family** before searching; no **side-by-side comparison**; no **"copy share link"** button despite shareable links being a headline feature.
- **Structural debt (maintainability):** `graph.js` is a **~1600-line god-file** spanning ~7 concerns (etymology-text formatting, family classification, the layout engine, perf/LOD/clustering, the vis.js lifecycle, trackpad/touch, the detail panel). Trackpad/touch/LOD logic is **duplicated** (~120 lines) between `graph.js` and `concept-map.js` ŌĆö exactly the sharing the abandoned `00007` proposed. `concept-map.js` re-attaches anonymous touch listeners on **every** reload (`:206`ŌĆō`:256`) without removing prior ones ŌĆö a real listener leak. Clustering is gated at `CLUSTER_MIN_NODES = 500` (`graph.js:872`), which etymology trees rarely reach, so that documented feature almost never fires.

## Part C ŌĆö Feature Roadmap toward a great etymology toolset

**Priority:** P0 foundational/high-leverage ┬Ę P1 high-value, moderate effort ┬Ę P2 larger/nice-to-have.
**Audience tags:** `[linguist]` `[hobbyist]` `[both]`.
A Ō£” marks items with a **draft spec created in this pass**.

### Theme 1 ŌĆö Data foundation (P0) `[both]`

- Ō£” **Native `descendants` adoption + inflection-aware search** ŌĆö `SPC-00014`. Replaces the capped, non-deterministic reverse lookup with Kaikki's curated `descendants` lists (completeness + R3/R4), and resolves inflected search queries to their lemma via `forms`/`form_of` (R6). Also folds in R2 (engine tests) and R7 (dead code). *This is the keystone: it fixes correctness and unblocks Themes 2ŌĆō3.*

### Theme 2 ŌĆö Flagship etymology features (P1)

- Ō£” **Doublets & shared-root pairs** ŌĆö `SPC-00015` `[both]`. Surface words that reach the *same* language from the *same* ancestor by different routes (inherited vs borrowed): *shirt/skirt*, *guard/ward*, *cattle/chattel*, *salt Ōåö salary*. A signature "wow" feature; the data already supports it via shared-ancestor reverse lookup.
- Ō£” **Sound-change correspondence view** ŌĆö `SPC-00016` `[linguist-lead, hobbyist-cool]`. Show IPA at each inheritance step and highlight the regular change (Grimm's law *pŌåÆf*, etc.), building on the existing IPA + Dolgopolsky tooling.
- **Semantic-shift panel** `[both]` (P1, future spec). Gloss at each ancestor stage, surfacing meaning drift; uses the per-stage `senses[].glosses` already fetched.
- **Word families & inflections** `[both]` (P1, future spec). Derivatives (`derived`/`related`) plus inflection tables (`forms`) for a word within its language; pairs naturally with `SPC-00014`'s `forms` work.

### Theme 3 ŌĆö Research rigor (P1ŌĆōP2) `[linguist]`

- Ō£” **Export & interoperability** ŌĆö `SPC-00017` `[linguist] + [hobbyist]`. Graph **image (PNG/SVG)** for sharing, plus structured export to **GraphML/GEXF** (Gephi), **Newick/Nexus** (phylogenetics tools), **CLDF** (Cross-Linguistic Data Formats), and plain JSON. Lets researchers take data out; lets hobbyists share a picture.
- **Provenance & citations** `[linguist]` (P2, future spec). Every edge traceable to its Wiktionary source, a "cite this" affordance, and a dataset/dump-version stamp for reproducibility.
- **Cognate-set builder + stable bulk API** `[linguist]` (P2, future spec). Rigorous, exportable cognate sets across a concept; a documented public API for programmatic access.

### Theme 4 ŌĆö Discovery & delight (P1ŌĆōP2) `[hobbyist]`

- Ō£” **Guided discovery** ŌĆö `SPC-00018` `[hobbyist]`. Word of the day, random / "surprise me", curated tours (Wanderw├Črter, Arabic loanwords, false friends), and "did you know" surprising-connection cards.
- **Quiz / game mode** `[hobbyist]` (P2, future spec). Guess-the-ancestor, cognate-match.

### Theme 5 ŌĆö Navigation & geography (P1ŌĆōP2) `[both]`

- Ō£” **Cheap navigation wins** (folded into `SPC-00019`): in-app history/breadcrumb trail; click-the-legend to isolate a family/era; filter-by-language/family; a "copy share link" button.
- **Geographic map view** `[both]` (P2, future spec). Plot languages / borrowing routes on a world map.
- **Side-by-side word comparison** `[both]` (P2, future spec).

### Theme 6 ŌĆö Polish & accessibility (P1) `[both]`

- Ō£” **Accessibility & UX baseline** ŌĆö `SPC-00019` (R8 above).

### Data-verification note

The Kaikki field-utilization figures in [A4](#a4--the-big-opportunity--unused-kaikki-data) are **code-derived**: they reflect which fields the ETL loader (`backend/etl/load.py`) ingests and which fields the service queries actually read ŌĆö *not* live `db.words` coverage counts, because MongoDB/Docker were not running during the audit. Before these numbers are cited externally, re-verify against a running stack, e.g.:

```js
// % of docs carrying each field
db.words.countDocuments({ descendants: { $exists: true, $ne: [] } })
db.words.countDocuments({ forms:       { $exists: true, $ne: [] } })
db.words.countDocuments({ derived:     { $exists: true, $ne: [] } })
db.words.countDocuments({ related:     { $exists: true, $ne: [] } })
```

The qualitative finding ŌĆö that these fields are present in the data yet unread by the app ŌĆö is confirmed by code reading and does not depend on the live counts.
